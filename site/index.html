Methodology

Understanding effective dimensionality and why it matters for every inference team.

1

The KV Cache Bottleneck

When a language model generates text, each attention layer stores two vectors per token: a key (what that token represents) and a value (what information it contributes). These key-value pairs are cached so they don't need to be recomputed for every new token. This is the KV cache.

The problem: the KV cache grows linearly with context length. At 8,192 tokens on a 32-layer model with 128-dimensional heads, a single user's KV cache consumes hundreds of megabytes. Serve 100 concurrent users, and you're dedicating tens of gigabytes of precious GPU memory purely to remembering what was already said.

Every major inference system — vLLM, TensorRT-LLM, SGLang — is fundamentally constrained by KV cache memory. Compressing it without destroying attention quality is the central challenge.

2

Not All Dimensions Are Equal

The standard assumption is that all 128 dimensions of a key vector are equally important. Random rotation methods like TurboQuant (Zandieh et al., ICLR 2026) exploit this assumption — they rotate vectors to spread energy uniformly, then quantize.

But this assumption is wrong. When you compute the covariance matrix of key vectors across many sequences and eigendecompose it, you find a dramatic spectral gap: a handful of dimensions carry almost all the variance, while the rest carry noise.

Across every model we've measured — Llama, Qwen, Mistral, Gemma — the effective dimensionality of key vectors is approximately 3–4% of the head dimension. For 128-dim heads, that's ~4 dimensions out of 128. The other 124 dimensions contribute almost nothing to attention scores.

3

The Measurement Pipeline

For each model, the measurement is straightforward:

  1. Run 100 calibration sequences from WikiText-103 through the model with caching enabled.
  2. For every (layer, head) pair, collect all key vectors and value vectors separately.
  3. Compute the uncentered covariance matrix: C = XTX / n, where X is the matrix of collected vectors.
  4. Eigendecompose: C = V diag(λ) VT. The eigenvalues λ₁ ≥ λ₂ ≥ … ≥ λd reveal how variance is distributed across dimensions.
  5. Compute the participation ratio (effective dimensionality):
Participation Ratio (Effective Dimensionality)
deff = (Σ λi)² / Σ(λi²)
If all eigenvalues are equal, deff = d (full dimensionality). If one eigenvalue dominates, deff ≈ 1. The participation ratio captures the smooth transition between these extremes.
  1. Compute the spectral gap: κ = λ_{d_eff} / λ_{d_eff+1} — the ratio between the last "signal" eigenvalue and the first "noise" eigenvalue. Higher κ means a sharper boundary between signal and noise.
  2. Report per-layer breakdowns, cumulative variance thresholds (dims for 95% and 99% variance), and representative eigenvalue spectra.

The entire pipeline runs in ~15 seconds per model on a single GPU. The covariance accumulation uses float64 for numerical stability.

4

Reading the Leaderboard

Key Signal % — primary metric
The fraction of head dimensions that carry meaningful signal in key vectors, normalized to head size: deff / head_dim. This is the number to compare across models, because it accounts for different head dimensions (128 for most models, 256 for Gemma). Lower = more concentrated = more compressible. A model at 2.5% is using 2.5% of its capacity for key selection — the other 97.5% is noise that can be safely compressed. Across all 14 models: 2.5% (Qwen 72B) to 4.5% (Mistral Small 24B). If you’re choosing a model for KV-compressed deployment, lower key signal % means cheaper inference.
Val Signal % — secondary metric
Same normalization for value vectors. Consistently 10–15× larger than keys — typically 30–50% of head dimensions carry meaningful variance. Values are fundamentally harder to compress. Any strategy that treats keys and values identically is leaving performance on the table.
Raw deff (key & value)
The unnormalized effective dimensionality: how many eigenvectors carry meaningful weight. For 128-dim heads, key deff ≈ 3–5. For Gemma’s 256-dim heads, key deff ≈ 7–8 — twice the number, same ~3% ratio. Always compare the normalized % across models, not raw deff, unless comparing models with identical head_dim.
κ (Kappa) — Spectral Gap
The ratio between the last “signal” eigenvalue and the first “noise” eigenvalue: λdeff / λdeff+1. Measures how sharp the cliff is between useful and useless dimensions. κ ≈ 1.2 means a gradual transition. κ > 100 (some Qwen models) means a near-total cliff — eigenvalues drop to essentially zero. Both work for compression, but very high κ models are especially suited to hard-threshold methods that simply discard all dimensions below deff.
The Key ≠ Value Asymmetry
The most actionable finding. Keys implement selection (“which tokens matter?”) with ~4 independent axes: recency, semantic similarity, syntactic role, domain. Values implement information retrieval (“what content to return?”) encoding dozens of linguistic and factual properties. Practical implication: compress keys to 2–3 bits, keep values at 3–4 bits. This asymmetric strategy, demonstrated in SpectralQuant (paper), achieves better quality than uniform compression at lower bitrate.
5

From Measurement to Compression

The spectral structure measured here directly enables SpectralQuant-style compression. Instead of treating all dimensions equally (as random rotation methods do), spectral-aware compression:

  1. Rotates vectors into their eigenbasis, sorting dimensions by importance
  2. Allocates more bits to the high-signal dimensions (the top deff)
  3. Allocates fewer bits — or skips error correction entirely — on the remaining ~96% noise dimensions

The result on Qwen 2.5-14B: cosine similarity 0.9485 vs TurboQuant's 0.9226 at 5.95× compression vs 5.02× — simultaneously better quality and better compression. This isn't possible without knowing the spectral structure first.

The leaderboard is the first step: measuring which models have exploitable structure, and how much.

Further reading: The full SpectralQuant method, theoretical analysis, and benchmark results are in the SpectralQuant repository and paper (Gopinath, NeurIPS 2026). The measurement code used for this leaderboard is at github.com/niashwin/spectral-index.

Why This Matters

The spectral structure of attention is a universal property with implications far beyond KV cache compression.

Inference Cost Reduction

Every major cloud provider spends billions on GPU inference. KV cache memory is the primary bottleneck for long-context serving. A 6× compression ratio from spectral-aware methods means serving 6× more concurrent users on the same hardware. At scale, this translates to hundreds of millions of dollars in infrastructure savings.

On-Device Intelligence

Mobile phones have 6–12 GB of shared RAM. A 7B model at 8K context consumes 2 GB just for the KV cache. With spectral compression, that drops to 333 MB — enabling meaningful long-context conversations on phones and edge devices. The spectral index tells deployment engineers exactly what compression ratio is achievable for their model.

Compression Strategy Selection

Different models have different spectral profiles. Some have sharp spectral gaps (high κ) that make aggressive compression safe. Others have gentler gradients that require more careful treatment. The leaderboard provides the data needed to make this decision before committing to a compression strategy.

The Keys ≠ Values Insight

The 10–15× asymmetry between key and value deff is a structural finding about how attention works, not just a compression trick. It suggests that the "selection" channel (keys/queries) operates in a fundamentally lower-dimensional space than the "information" channel (values). Understanding this asymmetry has implications for attention mechanism design, not just compression.

Cross-Modal Prediction

We've measured language models. But the same attention mechanism operates in vision transformers (ViT, DINOv2), audio models (Whisper), video transformers (VideoMAE), diffusion models (DiT), and protein structure predictors (ESMFold). We predict spectral gaps exist in all of them — with modality-specific deff values that reflect the intrinsic dimensionality of each domain's "relevance" question.

Open Research Questions

Does deff change during training? Is there a minimum deff for a given task complexity? Can you predict deff from architecture alone? Does fine-tuning change the spectral structure? Is the 3% ratio a consequence of optimization dynamics, or data structure? Every measurement on this leaderboard brings us closer to answering these questions.
